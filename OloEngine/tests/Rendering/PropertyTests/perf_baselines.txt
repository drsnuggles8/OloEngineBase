# Performance baselines for Layer-6 microbenchmarks.
#
# Format: one benchmark per line, "<key> <minimum_ns>".
# Lines starting with '#' are comments.
#
# Captured on the development machine — NOT a CI runner. These baselines
# are authoritative for regression detection on the same hardware; numbers
# will differ (sometimes by an order of magnitude) on other GPUs.
#
# To rebase (after confirming the change is expected):
#   set OLOENGINE_PERF_REBASE=1
#   run-tests-debug
# The test harness will rewrite this file with fresh numbers from the
# current run and pass unconditionally while the env var is set.
#
# Regression policy (see PerfRegressionTests.cpp). Enforcement is
# OPT-IN: by default both WARN- and FAIL-level regressions are RECORDED
# (gtest property + perf_history TSV) and logged via GTEST_LOG_(WARNING),
# but do NOT fail the test — these low-microsecond GPU microbenchmarks
# flake under machine/GPU contention (issue #324). Set
# OLOENGINE_PERF_STRICT=1 to turn a regression into an ADD_FAILURE
# failure on a quiet machine.
#   - Measured >= 2.5x baseline  -> FAIL-level (fails only when STRICT)
#   - Measured >= 1.5x baseline  -> WARN-level (fails only when STRICT)
#   - Faster than baseline       -> PASS, never fail on improvements
#
# Metric: minimum of 20 GL_TIME_ELAPSED samples after 5 warmup draws.
# The minimum (not median) is used because driver scheduling + GPU
# idle-ramp can only add time; the fastest run is the cleanest signal.
